SLIDE 2 : INTRODUCTION ET PROBLÉMATIQUE
----------------------------------------------------------------
Le contexte médical est critique. La pneumonie cause environ 2,5 millions de décès par an dans le monde.
Le diagnostic actuel repose sur l'analyse manuelle de radiographies thoraciques par des experts.
Les problèmes majeurs sont :
1. La lenteur du diagnostic dans les zones sous-effectif.
2. Le risque d'erreur humaine lié à la fatigue.
3. Le manque de radiologues qualifiés dans certaines régions.

Notre objectif est d'utiliser l'Intelligence Artificielle pour assister les médecins, en fournissant un pré-diagnostic rapide et fiable.

SLIDE 3 : OBJECTIFS DU PROJET MLOPS
----------------------------------------------------------------
Ce projet ne se limite pas à créer un modèle d'IA. Il s'agit de construire un système industriel complet (MLOps).
Nos objectifs sont :
1. Automatisation : Supprimer les tâches manuelles répétitives.
2. Reproductibilité : Garantir que chaque expérience est traçable.
3. Continuous Retraining : Le modèle doit s'améliorer seul avec le temps.
4. Déploiement : Rendre l'outil accessible via une application web publique.

SLIDE 4 : ARCHITECTURE GLOBALE
----------------------------------------------------------------
Nous avons mis en place une architecture modulaire basée sur des micro-services :
1. Gestion des Données : DVC (Data Version Control) pour versionner les datasets lourds.
2. Gestion du Code : Git et GitHub pour le versioning collaboratif.
3. Entraînement : PyTorch pour le Deep Learning.
4. Orchestration : Apache Airflow pour piloter les pipelines.
5. Tracking : MLflow pour enregistrer les métriques et modèles.
6. Production : Django pour l'interface utilisateur et Heroku pour l'hébergement cloud.

SLIDE 5 : LE MODÈLE D'INTELLIGENCE ARTIFICIELLE
----------------------------------------------------------------
Nous avons choisi l'architecture ResNet18.
C'est un réseau de neurones convolutifs (CNN) profond.
Nous utilisons le Transfer Learning : le modèle a été pré-entraîné sur ImageNet, puis adapté à nos radiographies.
Les entrées sont des images redimensionnées en 224x224 pixels.
La sortie est une classification binaire : "NORMAL" ou "PNEUMONIA".
L'optimiseur est Adam avec un learning rate de 0.001.

SLIDE 6 : LE DATASET
----------------------------------------------------------------
Nous utilisons le dataset "Chest X-Ray Images" (Kaggle).
Il contient 5,863 images.
La répartition est déséquilibrée : 73% de cas de pneumonie, 27% de cas normaux.
Nous avons divisé les données en :
- Train Set (89%) pour l'apprentissage.
- Validation Set (0.3%) pour le réglage.
- Test Set (11%) pour l'évaluation finale.

SLIDE 7 : CONTINUOUS RETRAINING (LE CŒUR DU SYSTÈME)
----------------------------------------------------------------
C'est l'innovation principale de ce projet.
Le système est capable de se mettre à jour automatiquement.
Le workflow est le suivant :
1. Surveillance : Airflow vérifie quotidiennement si de nouvelles données sont disponibles.
2. Déclenchement : Si oui, un réentraînement est lancé automatiquement.
3. Comparaison : Le nouveau modèle est comparé à l'ancien via MLflow.
4. Décision : Si le nouveau modèle est plus performant (Accuracy supérieure), il remplace l'ancien en production.

SLIDE 8 : INFRASTRUCTURE DOCKER
----------------------------------------------------------------
Pour garantir la stabilité, tout le système tourne dans des conteneurs Docker.
Nous avons 4 services qui communiquent entre eux :
1. PostgreSQL : Base de données pour Airflow et MLflow.
2. MLflow Server : Interface de tracking des expériences.
3. Airflow Webserver : Interface de gestion des pipelines.
4. Airflow Scheduler : Le moteur qui exécute les tâches planifiées.
L'utilisation de Docker assure que le projet fonctionne sur n'importe quelle machine (Windows, Linux, Mac).

SLIDE 9 : INTERFACE UTILISATEUR ET DÉPLOIEMENT
----------------------------------------------------------------
Pour l'utilisateur final (le médecin), nous avons développé une interface web avec Django.
Fonctionnalités :
- Accueil simple et professionnel.
- Module d'upload sécurisé pour les radiographies.
- Affichage instantané du résultat et du score de confiance.
L'application est déployée sur le cloud via Heroku (PaaS).
Elle est accessible partout via une URL publique sécurisée (HTTPS).

SLIDE 10 : RÉSULTATS ET PERFORMANCES
----------------------------------------------------------------
Sur le jeu de test, nous obtenons :
- Accuracy : 90%
- Precision : 88%
- Recall : 92% (très important pour ne pas rater de malades)
- F1-Score : 90%

Côté performance système :
- Temps de prédiction : moins d'une seconde par image.
- Temps de réentraînement : automatisé en tâche de fond.
